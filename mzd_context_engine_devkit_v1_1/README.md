# 毛泽东著作历史情境化阅读引擎：完整开发资料包

版本：`v1.1`  
文档状态：可进入原型研发  
产品归属：可验证的毛泽东数字思想博物馆

## 一句话定义

以用户选定版本的毛泽东著作为唯一正文，在用户进入、点击段落、选中文本或提出局部问题时，由预制阅读框架、结构化情境包与真实大模型工作流，把“历史背景、当时处境、思想与推理”实时流式填入固定界面槽位。

它不是重新做电子书，也不是在电子书旁边加聊天框，也不保留任何替代演示路径；唯一产品路径是“预制框架约束大模型，大模型实时生成相关流式输出”。

## 本资料包解决什么

本包用于产品、设计、内容、前端、后端、AI 工程、测试与运维共同启动项目，包含：

- 完整 PRD；
- 信息架构与界面槽位规范；
- 用户流程、产品流程和状态机；
- 系统架构、AI 运行时架构、部署架构；
- ContextPack 内容加工规范；
- 提示词注册表与声明式工作流；
- JSON Schema、OpenAPI、SSE 协议；
- PostgreSQL 数据库建表脚本；
- 质量评估、测试、安全、运维与迭代计划；
- 可运行的 FastAPI + 静态前端实时大模型骨架；
- 可编辑 DOT / Mermaid 图源与渲染后的 SVG / PNG；
- 示例数据、模板与校验工具。

## 已确认的产品边界

第一阶段坚持以下取舍：

1. 以项目方选定版本为唯一正文，不做版本异文比较。
2. 原始扫描影像不是 MVP 必需能力，后续可增量接入。
3. 证据链保持轻量：正文段落 ID、背景资料 ID、生成与提示词版本可追踪即可。
4. 原文始终居中；模型只能填充固定槽位，不能决定页面布局。
5. 历史材料只收录能解释文章问题意识、现实处境和思想推理的部分，不建设无边界通史百科。
6. 阅读端只保留 `framework_stream` 路径；所有解释内容由真实大模型在预制框架内实时生成。

## 快速阅读顺序

1. `PRD.md`：先确认产品目标、范围和验收标准。
2. `docs/02_UI_IA.md`：确认界面框架与交互槽位。
3. `docs/05_AI_PROMPTS_WORKFLOWS.md`：确认核心创新——提示词与工作流。
4. `docs/06_CONTEXTPACK.md`：确认历史背景层如何加工。
5. `ARCHITECTURE.md`：确认服务、数据和运行时架构。
6. `docs/14_ROADMAP_BACKLOG.md`：进入排期与研发拆分。
7. `starter/README.md`：启动实时大模型骨架。

## 建议的第一条纵向切片

不要一开始处理全集。首个可交付原型只需要：

- 1 篇文章；
- 8–15 个阅读锚点；
- 30–60 个情境单元；
- 3 个固定槽位；
- 1 套固定三槽位预制框架；
- 1 套内容编辑与审核流程；
- 真实大模型实时流式生成路径跑通；
- 20–30 条 Golden Set 测试题。

其完成标准是：用户读到任一锚点时，能在不离开原文的情况下理解“此前发生了什么、当时难在哪里、这段如何形成判断”，并且内容可以稳定复现、审核和修订。
