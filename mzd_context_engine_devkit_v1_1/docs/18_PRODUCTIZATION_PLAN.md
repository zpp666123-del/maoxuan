# 18. 产品化推进计划

## 当前产品形态

- 阅读主界面采用中心正文，工具按钮按需打开弹窗。
- 后端只走真实大模型与真实图像接口；缺配置时明确失败。
- 地图由本地 Leaflet 组件和文章变量即时生成，AI workflow 负责后续变量增强，并以 `map.variables.committed` 事件刷新变量层。
- 默认开启后台预热：配置 API 后进入文章会自动准备全篇总览和地图变量，不打开等待弹窗。
- 时间线、人物、事件、插图由 workflow/agent/subagent 驱动。
- 毛选正文从本地静火版本语料目录加载。

## 最近里程碑

### M1：可恢复的单机产品

- workflow artifact 落入 SQLite；
- 重启后同一文章/锚点/工作流可复用历史真实生成结果；
- `forceRegenerate` 明确绕过缓存；
- 默认开启“进入文章后自动准备全篇与地图”，后台 workflow 完成后自动回到 ready；
- 地图打开即显示变量地图，关键词可即时加入，不等待模型；
- `map_context` 的真实大模型结果可持久化并复播 `map.variables.committed`；
- 本地设置页覆盖文本模型和图像模型配置。

### M2：可运营的产品内测

- 增加 run history 查询；
- 增加模型错误、成本和生成次数统计；
- 增加文章级首次总览预热；
- 增加用户反馈入口。

### M3：生产化后端

- SQLite payload 迁移到 PostgreSQL；
- 引入后台任务队列和并发限制；
- 增加 API key 安全存储；
- 增加审计日志和发布回滚。

## 验收门槛

- 未配置 API 时无伪内容输出；
- 配置 API 后所有按钮都能触发对应 SSE workflow；
- 重复点击默认复用真实生成结果；
- 强制重新生成会重新调用模型；
- 桌面和移动端无横向溢出；
- `tools/validate_package.py` 全通过。
