# 12. 安全、权限、内容权利与合规设计

## 12.1 安全目标

- 防止未授权修改或发布历史内容；
- 防止用户输入和资料内容注入模型规则；
- 防止模型生成伪造来源、坐标、日期和关系；
- 保护用户会话、问题和反馈；
- 记录关键内容与配置变更；
- 对权利不清的正文和资料进行细粒度可见性控制；
- 保证模型供应商故障或泄露风险不影响规范资料。

## 12.2 权限模型

角色：

| 角色 | 能力 |
|---|---|
| anonymous_reader | 阅读公开文章和运行记录、临时交互 |
| registered_reader | 保存进度、笔记、反馈 |
| content_editor | 编辑草稿文章、ContextUnit、来源和链接 |
| historical_reviewer | 审核历史事实、时间和来源 |
| thought_reviewer | 审核原文关联、思想与推理 |
| publisher | 创建 Release、回滚 |
| ai_admin | 管理 Prompt、Workflow、模型策略和评测 |
| rights_admin | 管理正文与资料权利状态 |
| system_admin | 用户、环境、密钥、审计和运维 |

关键原则：编辑者不能独自发布；AI 管理员不能直接修改正文；系统管理员默认不能绕过内容审核。

## 12.3 认证与会话

- 公共阅读允许匿名短期会话；
- 管理端必须使用强认证与多因素认证；
- 管理端会话短时、支持强制失效；
- Access Token 最小权限；
- Service Account 分环境；
- SSE 使用短期签名或同源 Cookie；
- 不在 URL 中放用户问题或敏感 Token。

## 12.4 Prompt 注入防护

### 输入隔离

- 系统 Prompt、开发者 Prompt 与数据内容分层；
- 用户选区、问题、来源摘录都用显式数据边界；
- 不将用户问题拼接到系统指令；
- 模型没有任意工具调用权限；
- 模型只能选择已批准 ID。

### 输出限制

- JSON Schema；
- ID 白名单；
- 日期/坐标/关系回填；
- HTML 转义；
- 禁止模型返回 URL 作为自动请求；
- 禁止模型决定权限、发布和删除。

### 内容导入

外部资料可能包含指令性文字。导入处理必须把所有内容视为非可信数据，移除脚本、宏和嵌入式活动内容。

## 12.5 模型数据安全

- 只向模型发送当前任务最小必要正文和资料；
- 权利受限资料可配置禁止发送外部模型，只允许自托管或人工处理；
- API 日志不记录完整敏感文本；
- 模型请求启用传输加密；
- 模型供应商密钥仅存密钥管理系统；
- 为不同环境使用不同项目和配额；
- 定期审查供应商的数据保留设置；
- GenerationRun 保存哈希和必要摘要，原始 Prompt 可按策略加密或缩短保留期。

## 12.6 用户隐私

本产品无需收集用户真实身份即可提供核心体验。默认不：

- 推断或保存用户政治倾向；
- 根据阅读内容建立敏感人物画像；
- 将用户问题用于模型训练；
- 向第三方广告系统发送阅读段落；
- 公开显示个人反馈。

可收集的最小数据：匿名会话、阅读进度、交互类型、性能、卡片反馈。用户可删除账户、笔记和可关联会话数据。

## 12.7 内容权利

即使不做原始影像，仍需登记：

- 选定版本正文来源；
- 是否允许公开全文；
- 背景资料的引用或摘要权限；
- 地图、图片和外部资产权利；
- 人工整理内容的许可；
- 导出和 API 使用范围。

`rights_status`：

- `public`
- `licensed_public`
- `internal_only`
- `restricted_excerpt`
- `unknown`

运行时检索必须考虑可见性。权利不清的 Source 可以支持内部编辑，但不能把受限原文直接展示给公共用户。

## 12.8 审计

必须审计：

- 正文导入和修订；
- 锚点变化；
- ContextUnit 增删改；
- 来源等级与权利变化；
- Prompt/Workflow 发布；
- 模型策略变化；
- 审核决定；
- Release 发布/回滚；
- 管理员权限变化；
- 批量导出。

审计记录不可由普通管理员修改，需定期备份。

## 12.9 内容安全与呈现伦理

- 历史苦难、战争和暴力以解释性方式呈现；
- 处境推演不得将真实伤亡简化为娱乐积分；
- 不设计以“扮演历史人物发号施令”为核心的拟人聊天；
- 不把 AI 解释冒充作者本人发言；
- 有争议时保留事实、解释和不确定性的差别；
- 系统不根据用户立场改变资料和历史结果。

## 12.10 Web 安全

- CSP；
- CSRF 保护；
- XSS 转义；
- SQL 参数化；
- 上传文件类型和大小限制；
- 病毒/恶意文件扫描；
- SSRF 防护，模型输出 URL 不自动抓取；
- Rate Limit；
- 管理端 IP/设备策略可选；
- 依赖与镜像扫描；
- 生产数据库不直接暴露公网。

## 12.11 事件响应

P0 安全事件：

- 管理权限泄露；
- 公开发布受限正文；
- Prompt 注入导致规则泄露或数据越权；
- 审计被篡改；
- 用户问题或身份泄露。

流程：隔离 → 回滚/撤权 → 保全审计 → 影响评估 → 修复 → 通知 → 复盘 → 增加测试。
