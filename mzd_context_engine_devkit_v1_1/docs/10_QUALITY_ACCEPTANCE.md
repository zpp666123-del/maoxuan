# 10. 质量评估、发布门禁与验收标准

## 10.1 质量不是单一“正确率”

本产品的输出质量至少包含八个维度：

1. 与当前原文的相关性；
2. 历史事实准确性；
3. 时间边界；
4. 三槽位语义分离；
5. 推理清晰度；
6. 来源覆盖；
7. 信息密度与可读性；
8. 不确定性表达。

## 10.2 评分量表

每项 1–5 分。

### 相关性

- 5：直接解释当前锚点，不含无关信息；
- 3：总体相关但有泛化；
- 1：与当前文本关系弱或答非所问。

### 历史准确性

- 5：具体事实均与批准资料一致；
- 3：核心正确但有不必要概括；
- 1：出现错误或无依据具体事实。

### 时间边界

- 5：严格区分此前、当时与后来；
- 3：语言略含后见倾向但未影响结论；
- 1：用后续结果作为当时依据。

### 槽位分离

- 5：背景、处境、思想各司其职；
- 3：少量重复；
- 1：三卡基本是同一摘要。

### 推理清晰度

- 5：问题、依据、判断和方法关系清楚；
- 3：有结论但链条不完整；
- 1：只给标签或口号。

### 来源覆盖

- 5：所有具体事实均有 sourceIds；
- 3：少量解释性内容未明确；
- 1：大量事实不可追踪。

### 信息密度

- 5：短而有用，扫读即可理解；
- 3：稍长或略空；
- 1：冗长、重复或过于简单。

### 不确定性

- 5：事实、解释、推断和未知区分清楚；
- 3：有有限模糊；
- 1：把推断写成事实。

## 10.3 自动门禁

流式运行记录必须全部通过：

- JSON Schema；
- source ID 存在；
- source 可见性允许；
- 日期不越界；
- 卡片长度；
- points 数量；
- 禁止 HTML/脚本；
- 卡片间重复率低于阈值；
- 场景 ID、日期、坐标来自 ContextPack；
- 场景节点上限；
- Anchor 相关度下限；
- 所有 P0 锚点有运行记录。

## 10.4 人工门禁

每篇文章发布至少完成：

- 文章级定位审核；
- 锚点划分审核；
- 所有高风险具体事实审核；
- 所有“回到当时”处境卡审核；
- 所有动态场景审核；
- 思想卡与原文一致性审核；
- 抽查背景卡与来源；
- 整体连续滚动体验审核。

MVP 首篇建议全量人工审核；规模扩大后再按风险抽样。

## 10.5 发布阈值

建议阈值：

- 平均总分 ≥ 4.0/5；
- 任一 P0 运行记录无单项 1 分；
- 时间边界平均 ≥ 4.5；
- 历史准确性平均 ≥ 4.5；
- 无依据具体事实率 < 1%；
- 时间泄漏率 < 2%；
- Schema 合法率 100%；
- 场景数据合法率 100%；
- 关键 Golden Set 通过率 100%；
- 全量 Golden Set 通过率 ≥ 90%。

阈值是项目基线，可随真实评测调整，但任何降低需记录 ADR。

## 10.6 Golden Set 结构

```yaml
id: gs_selection_001
articleId: article_x
anchorId: anchor_03
event:
  type: text_selected
  selection: "..."
mode: time_frozen
expected:
  updateSlots: [thought]
  keepSlots: [background, situation]
  allowedScene: [decision, none]
  mustMentionConcepts: ["..."]
  forbiddenClaims: ["后来发生的结果"]
  allowedSourceIds: [src_01, src_04]
  expectedUnavailable: false
rubric:
  relevance: 5
  timeBoundary: 5
```

## 10.7 线上质量监控

实时统计：

- Validator fail；
- repair rate；
- unavailable rate；
- 用户“不准确”反馈；
- 用户“与本段无关”反馈；
- 用户“重复”反馈；
- 模型/Prompt/文章维度的错误聚合；
- 低来源覆盖率；
- 动态结果与流式运行记录差异过大；
- 单锚点异常高提问率，可能表示内容不足。

## 10.8 质量问题分级

### P0

- 严重历史事实错误；
- 后见结果被写成当时已知；
- 伪造来源；
- 原文被改变；
- 权利受限内容泄露；
- 提示注入改变系统规则。

处理：立即下线相关运行记录/动态能力，回滚发布。

### P1

- 核心处境遗漏；
- 思想推理与原文明显不一致；
- 场景关系错误；
- 三槽位大面积重复。

处理：24 小时内修订或锁定人工审核运行记录。

### P2

- 表达冗长；
- 个别来源显示不清；
- 视觉标签拥挤；
- 次要背景缺失。

进入正常迭代。

## 10.9 用户研究验收

首轮至少 8–12 名目标用户完成：

- 不看说明进入文章；
- 解释某锚点的前因、处境和思想；
- 切换回到当时；
- 使用一种场景；
- 对选区进行深挖；
- 返回原文继续阅读。

关注：

- 是否理解三槽位差异；
- 是否觉得 AI 抢走原文注意力；
- 是否能判断哪些是当时信息、哪些是后来；
- 是否感到内容太多或太浅；
- 是否愿意继续读下一锚点。
