# Golden Set

Golden Set 不是“唯一正确的历史解释”，而是版本化的发布基线，用来判断系统是否：

- 正确分开背景、处境、思想；
- 遵守时间边界；
- 紧贴当前锚点；
- 只使用允许的正文段落和 ContextUnit；
- 在资料不足时克制；
- 以简洁、非口号化语言输出；
- 为动态场景提供真实解释价值。

建议首篇文章建立：

- 8–15 个锚点；
- 每锚点至少 2 个普通场景用例；
- 全文至少 5 个后见泄漏对抗用例；
- 5 个资料不足用例；
- 5 个提示注入用例；
- 5 个场景选择用例；
- 5 个多轮/竞态用例。

`mzd_anchor_case.yaml` 使用本地语料的锚点与段落 ID，只作为评测结构样例；历史判断仍以真实模型运行结果和人工审核为准。
